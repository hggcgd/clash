## 功能介绍

- 轻量内嵌核心：进程内直接调用，低延迟、无需额外进程与端口层面的 HTTP 代理交互。
- 托盘快捷管理：提供系统托盘菜单，支持一键开启/关闭系统代理、自动启动、打开面板、管理员重启、TUN（游戏）模式切换等。
- 订阅管理：自动生成并重载 `config.yaml`，方便新用户，自动重写配置文件。
- 适合国情，默认国内各机构不再通过代理，而影响登录各种国内的APP。
  
## 系统要求

- 操作系统：Windows 10/11 x64。
- 运行权限：普通用户可用；启用 TUN（游戏模式）需管理员权限。
- 网络环境：确保本机防火墙允许本程序访问网络与本地端口。

## 安装与打包

- 安装：无需安装，解压 `class.zip` 至任意文件夹，保持 `clash.exe` 与 `clash.dll` 位于同一目录。
- 首次运行：双击 `clash.exe`，系统托盘会出现图标，默认使用内置 `config.yaml`（资源中加载），可直接使用或在订阅管理中自定义。

## 使用说明

- 启动与退出：
  - 运行 `clash.exe` 后托盘出现图标；右键菜单选择“退出”即可关闭并清理系统代理。
- 订阅管理（代理源）：
  - 托盘右键菜单选择“订阅管理”，添加名称与 URL（以 `http://` 或 `https://` 开头）。保存后自动生成并重载 `config.yaml`。
- 系统代理：
  - 菜单“系统代理”用于开启，启动时检测到节点有效，自动设置系统代理。

## 默认配置要点（可在运行目录自动生成的 `config.yaml` 中查看/调整）

- 默认面板密码：`secret: "666666"`，用于访问面板与 API；可自行修改。
## 常见问题与排查

- 程序提示“无法加载 clash.dll”：确认 `clash.dll` 与 `clash.exe` 位于同一目录，且未被安全软件拦截。
- 网络异常或无法代理：在托盘菜单重新开启“系统代理”，或检查是否开启了 TUN 模式；必要时重启程序或清除系统代理后再试。
- 端口冲突：关闭同类软件（如 sing-box/xray 等）；程序会在异常时尝试清理系统代理并重启核心。
- 权限相关：启用 TUN 需要管理员权限；若被 UAC 拒绝，可先退出再以管理员身份运行。

## 安全与隐私
- 程序退出时会尝试清除系统代理，避免残留影响。
